# Learning theory

## Summary

Also called **ML strategy** - knowing *what to do next*. The same algorithm produces very different results across teams; the difference is strategy.

Decisions a practitioner faces: collect more data (and *which* data), switch algorithms, buy more compute / train longer, change architecture. The goal is to turn ML from "black magic / tribal knowledge" into a systematic engineering discipline - e.g. don't just delete the lines with syntax errors, and profile before you optimize.
